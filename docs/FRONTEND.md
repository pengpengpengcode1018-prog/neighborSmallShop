# 前端与交互护栏

## 双端边界

- `miniapp/` 面向微信小程序，使用 uni-app Vue 3、Pinia、Sass 和 Vant Weapp；Vant Vue 不得引入。
- `admin-web/` 面向桌面浏览器，使用 Vue 3、Element Plus 和 ECharts。
- 两端共享 API 契约和产品语言，不共享依赖 DOM 或微信原生能力的 UI 组件。

## 交互原则

- 先让配送范围、价格、库存、时段和订单状态可理解，再做装饰。
- 每个网络界面明确 loading、empty、success、error、retry；危险动作二次确认。
- 金额统一两位小数；状态名称来自集中映射，不在页面散落字符串。
- 表单错误靠近字段，保留用户可安全重试的输入。
- 小程序底部安全区、触控尺寸和弱网恢复属于验收条件；后台保留键盘和焦点可用性。

## 组件方向

- 小程序沉淀 `StoreCard`、`ProductCard`、`CartBar`、`OrderCard`、`EmptyState`。
- 后台使用统一列表页、筛选栏、编辑抽屉和状态标签，不复制业务表单。
- 主题变量集中在各端样式入口；页面不得重写组件库内部选择器来完成常规布局。

## 验证

- 管理后台关键旅程用浏览器自动化检查 DOM、console、网络失败和截图。
- 小程序至少通过编译，并在微信开发者工具或真机验证登录、支付和订阅消息相关功能。
- 视觉证据链接到执行计划；修复回归时追加稳定重现步骤。
