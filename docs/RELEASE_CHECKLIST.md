## 微信隐私保护指引填报基线

入口：微信公众平台 -> 账号设置 -> 服务内容声明 -> 用户隐私保护指引 -> 去完善。只声明当前正式体验版实际发生的行为，平台保存后等待配置同步，再进行真机复核。

| 项目                       | 按当前实现如实填写                                                                                                                                                                                                     | 不应填写/注意事项                                                                                                                                                           |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 用户信息（微信昵称、头像） | 微信身份登录后，居民可主动选择微信头像并填写昵称，用于在本人登录区展示个人资料，以及帮助平台管理员在居民列表中识别账号。该资料为可选项，跳过或拒绝不影响浏览、购物和下单。                                             | 不写“登录时自动获取”；当前实现不调用静默用户资料接口，只保存居民主动选择或填写的内容。头像通过 API 域名返回，不能在日志、截图或后台接口中返回图片二进制。                   |
| 收集你的手机号             | 居民主动点击微信原生手机号授权按钮后，用于绑定当前微信账号、提供购物与订单服务、在新建收货地址时预填本人联系手机号，以及必要的订单履约联系。拒绝授权仍可浏览店铺，但购物车、地址、订单和支付等个人能力需要登录及绑定。 | 不写“静默获取”；手机号只能来自微信一次性动态 code，不能把地址表单中的号码反写成已验证账号手机号。                                                                           |
| 收货人和地址信息           | 居民在地址表单中手工填写收货人、联系手机号、小区、楼栋、单元、门牌及详细地址，用于校验配送范围、保存本人收货地址、生成订单履约快照和完成配送。                                                                         | 当前代码没有调用 `wx.chooseAddress`、定位或地图接口；若平台条目特指该接口，不得为了“看起来完整”而误报。后台列表只显示脱敏手机号，完整订单地址仅在已鉴权履约详情中按需展示。 |
| 订阅消息                   | 居民在具体通知场景中主动授权，用于发送支付、接单、制作、配送和退款进度；授权结果只影响通知，不改变订单、支付或退款真实状态。                                                                                           | 订阅消息是单独的微信授权，不要把它误写成手机号或地址私密接口；未配置真实模板前继续保持发布 `NO-GO`。                                                                        |

真机复核步骤：

1. 保存配置后等待平台同步；用新微信账号或清除授权状态，从登录入口重新进入。
2. 确认跳过昵称/头像完善仍可正常使用；主动选择头像并填写昵称后，首页与后台居民列表展示一致，重新进入小程序后资料仍可恢复。
3. 在微信公众平台把 `https://api.surroundsmallshops.com` 同时配置为 `request` 与 `downloadFile` 合法域名，真机确认居民头像可以显示；不得关闭域名校验绕过该检查。
4. 确认隐私提示在手机号授权前出现；拒绝后仍可浏览，接受后手机号绑定成功。
5. 新建收货地址，确认联系手机号自动预填且允许本人修改；编辑已有地址不得被账号手机号覆盖。
6. 从小程序右上角“...”打开隐私保护指引，逐项核对展示文本与上表一致。
7. 保存只包含声明项目、审核状态和时间的截图；不得包含完整手机号、头像内容、地址、OpenID、密钥或支付标识。
8. 提交正式审核时再次核对审核信息表中的隐私声明；平台审核通过前不得把本门禁改为 `GO`。

## 发布和回滚顺序

1. 冻结版本，保存 `npm run verify`、`npm run release:audit` 和 `npm run release:rehearse` 的同一提交证据。
2. 通过密钥系统注入生产变量；只向一次性迁移容器注入 DDL+DML 迁移账号执行 `prisma migrate deploy`，常驻 API 只接收 DML 应用账号；确认 `/api/v1/ready` 后再切流。
3. 先开放后台只读检查，再进行微信体验版黄金旅程；全部人工门禁签字后才提交正式审核或切换流量。
4. 应用回滚只切回上一镜像，不自动回滚数据库迁移；发生支付/退款结果未知时保留订单并按商户单号查询，不重复发起外部动作。
5. 回滚后复核回调积压、关单租约、通知 `UNKNOWN`、库存和支付/退款对账，保存去敏事件编号。
