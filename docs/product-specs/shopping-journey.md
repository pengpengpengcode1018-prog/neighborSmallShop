# 居民购物旅程

来源：需求说明书第 6-21、35-36、39-42、47 页。

## 起始条件

- 游客首次进入可浏览营业店铺；准备结算前必须登录并选择一个平台启用的小区。
- 已选择小区时，只有可配送至该小区且处于营业状态的店铺可进入可下单状态。

## 小区选择与恢复

- 公共小区列表只返回启用且未删除的固定小区，不使用地图定位或距离排序。
- 游客选择保存在本地；登录居民选择由服务端保存，并同步到本地用于恢复浏览上下文。
- 登录后若账号已有有效服务端小区，以服务端选择为准；只有服务端为空时，才把有效的游客本地选择同步给账号，避免静默覆盖跨设备选择。
- 登录状态切换小区时，服务端保存成功后才更新本地；网络失败保留原选择并允许重试。
- 已选小区被停用或删除后，恢复时清除该选择并提示重新选择，不继续用于配送范围判断。

## 店铺浏览与可见性

- 未选择小区时，游客可浏览所有未删除且状态为 `OPEN` 或 `PAUSED` 的店铺；此时仅展示默认起送额、配送费和预计时效，并明确标记“选择小区后确认配送”，不得返回可下单状态。
- 选择小区后，先确认小区仍为启用且未删除，再只展示与该小区存在 `ACTIVE` 配送关系的未删除店铺；暂停配送关系、禁用店铺和已删除店铺均隐藏。
- 店铺状态为 `PAUSED` 时仍可出现在列表并进入详情，但必须展示暂停接单状态，且 `canOrder` 为 `false`；只有 `OPEN` 店铺在有效小区配送关系下可下单。
- 店铺列表支持按店铺名称进行去除首尾空格后的包含搜索，使用后台维护的排序值优先、创建时间次序分页返回。
- 小区配送关系可覆盖店铺默认起送额、配送费和预计时效；覆盖合并只在服务端完成，金额统一返回两位小数字符串。
- 店铺详情沿用列表的可见性规则。指定有效小区但店铺无有效配送关系时返回稳定的 `STORE_NOT_DELIVERABLE`；不存在、禁用或已删除店铺返回 `STORE_NOT_FOUND`。
- 店铺列表展示后台上传的可选 Logo，店铺详情展示可选封面；没有图片时使用文字占位。图片只使用服务端生成的受控媒体 URL，不接受居民端传入或拼接外部地址。
- 店铺公开详情同时返回当前启用的预约送达时刻与截止时间，供订单确认页选择；是否满额仍以订单预览和创建事务的实时容量校验为准，客户端列表不构成可下单承诺。

## 商品浏览与可见性

- 店铺商品区只返回启用且未删除分类；分类按后台排序值优先、创建时间次序展示，停用或已删除分类及其商品不从居民接口旁路出现。
- 商品列表只返回未删除且后台状态为 `ON_SALE` 或 `SOLD_OUT` 的商品；`OFF_SHELF` 商品和软删除商品在列表、分类筛选与详情中均隐藏。
- 后台状态为 `SOLD_OUT`，或实时库存为零的商品，居民端统一显示为 `SOLD_OUT` 并返回 `canPurchase: false`；客户端不单独根据缓存库存推导购买资格。
- 只有商品显示状态为 `ON_SALE`、库存大于零，并且当前店铺在所选小区内可下单时，`canPurchase` 才为 `true`。未选小区或店铺暂停接单时仍可浏览商品，但不可购买。
- 分类筛选必须属于当前店铺且当前可见；跨店、停用或已删除分类返回稳定的 `CATEGORY_NOT_FOUND`，不静默切换店铺上下文。
- 商品价格和可选原价统一返回两位小数字符串；列表公开图片、名称、描述、价格、销量、库存与状态，详情追加轮播图、正文和售后说明，不返回后台排序、库存预警或备注字段。
- 商品列表以主图作为缩略图；详情先展示主图，再按后台顺序展示最多 6 张轮播图并对重复 URL 去重。未上传图片时保持明确占位，不影响商品浏览和购买资格。
- 商品详情复用所属店铺的可见性和配送上下文；下架、软删除或隐藏分类下商品返回 `PRODUCT_NOT_FOUND`。

## 单店铺购物车

- 购物车属于登录居民并保存于服务端；所有购物车接口重新验证居民会话，管理员 token 和其他居民均不能读取或修改该购物车。
- 一个居民购物车同时只能绑定一家店铺。向同店重复加入同一商品时增加数量；向另一店铺加购先返回 `CART_STORE_CONFLICT`，只有客户端二次确认后提交 `replaceExistingCart: true`，服务端才在事务中原子清空原项并加入新商品。
- 加入和修改数量必须为正整数；最终数量不得超过商品实时库存或限购数量。无库存返回 `PRODUCT_STOCK_NOT_ENOUGH`，超限购返回 `PRODUCT_PURCHASE_LIMIT_EXCEEDED`。
- 购物车写入使用账号当前有效小区，并重新校验店铺配送关系、营业状态、商品/分类上下架和实时库存；未选择小区、店铺不可配送/暂停或商品失效时拒绝写入，不信任客户端传入的价格与资格。
- 读取购物车时不静默删除后来下架、售罄、超库存、店铺暂停或小区切换后失效的项目，而是返回项目和购物车级不可用原因，允许用户删除、清空或恢复条件。
- 商品单价、小计、商品总额、配送费和预计应付金额均由服务端使用十进制定点数实时计算并返回两位小数字符串。购物车金额是当前展示，不锁价也不锁库存，订单预览仍必须重算。
- 服务端同时返回当前小区覆盖后的起送额、距起送差额、是否达到起送额和 `canCheckout`；本切片只展示结算资格，不提前创建订单。
- 删除单项后若购物车为空，删除空购物车记录；清空购物车是显式的幂等操作。

## 收货地址

- 收货地址属于登录居民并保存于服务端；居民只能读取、修改、设默认和删除自己的未删除地址，其他居民访问时统一返回 `ADDRESS_NOT_FOUND`，不泄露地址是否存在。
- 地址包含收货人姓名、联系手机号、平台小区、楼栋、可选单元、门牌号、可选补充地址、标签和默认状态；标签固定为家、公司、学校、其他，用户不能手动创建平台不存在的小区。
- 新增地址时，如果当前居民已绑定微信手机号，表单首次初始化自动把该号码填入“联系手机号”；该值只是默认值，居民可以改为实际收货人的其他有效手机号。编辑已有地址时必须保留地址原联系电话，不得被账号手机号覆盖。
- 当前居民尚未绑定手机号时，从新增地址入口先触发显式微信手机号授权；授权成功后返回地址表单并自动填充，拒绝或失败时不创建地址并提供重试入口。
- 收货人、手机号、小区、楼栋和门牌号必填；手机号按中国大陆 11 位手机号校验，所有文本在边界去除首尾空格并限制长度，完整手机号不进入日志、截图或文档证据。
- 同一居民可保存多个地址，但最多一个默认地址。首个地址自动成为默认；新建时显式设默认或调用设默认接口时，在同一事务中撤销旧默认并设置新默认，数据库唯一键阻止并发后出现多个默认地址。
- 删除采用软删除，历史订单保存独立地址快照，不读取被修改后的地址。删除默认地址后，优先把 `lastUsedAt` 最近的剩余地址设为默认；尚无使用记录时以最近编辑时间作为确定性回退。
- 小区后来停用或删除时不静默删除地址；地址列表仍返回该地址并标记 `available: false`、`unavailableReason: COMMUNITY_UNAVAILABLE`，允许用户编辑到有效小区或删除，但不能再把该地址设为默认。
- 地址列表按默认地址优先、最近编辑时间排序。地址写入只接受启用且未删除的小区，否则返回 `COMMUNITY_NOT_FOUND`；后续订单预览仍必须重新校验地址小区是否在当前店铺配送范围内。

## 旅程

1. 首页展示当前小区、轮播/公告、可配送店铺和搜索入口。
2. 用户查看店铺营业状态、起送金额、配送费与预计时效。
3. 店铺页按分类展示单规格商品；下架、售罄商品不可加入购物车。
4. 购物车首次加入商品时绑定店铺；切换店铺需明确确认清空原购物车。
5. 用户选择属于当前配送小区的有效地址和可用配送时段。
6. 订单预览由服务端重新读取价格、库存、配送费和起送规则。
7. 用户以唯一请求 id 创建订单，随后进入微信支付。
8. 用户从订单列表进入快照详情查看配送信息和状态时间线；待付款阶段可取消并由服务端原子恢复库存一次。
9. 已支付未接单订单可选择原因申请一次整单退款；申请后查看审核和微信处理进度，只有可信退款成功才显示已退款。

## 可观察结果

- 数量、商品小计、配送费和应付总额始终可解释。
- 首次手机号授权成功后，新建地址无需重复输入本人手机号；修改地址联系电话不会反向覆盖账号绑定手机号。
- 地址不在范围、店铺不可接单、库存不足、未达起送额和时段满额均阻止下单，并给出恢复动作。
- 客户端缓存价格不构成订单依据；服务端结果变化时要求用户确认最新金额。

## 失败与恢复

- 网络失败保留安全的本地选择并允许重试，不重复创建订单。
- 无店铺/无商品/购物车为空/无地址/无时段使用明确 empty state。
- 登录过期回到登录流程后，能恢复不涉及敏感数据的浏览上下文。
