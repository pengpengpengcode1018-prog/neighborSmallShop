# 订单、库存与配送状态

来源：需求说明书第 15-20、27-36、50-51 页。

## 状态机

正常路径：

`pending_payment -> paid -> accepted -> preparing -> waiting_delivery -> delivering -> completed`

取消/退款路径：

- `pending_payment -> cancelled`
- `paid -> refund_pending -> refunded`
- 第一版居民退款只允许 `paid -> refund_pending`；审核拒绝时显式回到 `paid`，审核通过并收到可信微信退款成功后进入 `refunded`。`accepted` 后退款留给后续售后范围。
- `completed`、`cancelled`、`refunded` 为终态。

任何未列出的跳转都返回 `INVALID_ORDER_STATUS`，不修改数据。

## 创建与库存

- 创建订单在一个数据库事务中校验配送/营业/起送/时段/库存，执行带库存下限条件的更新，写订单、商品快照和首条状态日志。
- 订单预览和创建只接收地址、配送选择、备注及创建请求 id，不接收客户端价格、总额或库存资格；两次操作都从当前服务端购物车重算，创建时再次执行全部校验。
- 请求 id 对用户唯一；同一用户以相同请求 id 和相同参数重试时返回原订单，不重复扣库存；相同请求 id 携带不同参数返回 `DUPLICATE_REQUEST`，不产生第二个订单。
- 商品和地址保存快照，历史订单不受后续编辑影响。
- 取消或 15 分钟超时恢复库存；`stock_released` 保证只恢复一次。存在微信支付单时必须先在数据库短事务中领取关闭任务，再离开事务查单/关单，最后用订单行锁提交终态，禁止持锁等待微信网络。
- 创建待支付订单时按商品执行 `stock >= quantity` 的条件扣减；任一商品失败时订单、订单项、状态日志、库存、地址最近使用时间和购物车清理全部回滚。
- 订单创建成功后清空当前购物车并更新地址最近使用时间；订单金额、库存和地址真相已经进入快照，不再依赖购物车或可变主数据。

## 订单预览与配送

- 预览与创建请求包含 `addressId`、配送类型 `ASAP|SCHEDULED`、可选预约日期/时段、最长 200 字备注；预约配送必须同时提交日期和时段，尽快送达不得携带时段。
- 地址必须属于当前居民、未删除且所属小区有效，并处于购物车店铺的有效配送范围；否则分别返回 `ADDRESS_NOT_FOUND` 或 `STORE_NOT_DELIVERABLE`。
- 店铺必须未删除、状态为营业且启用所选配送方式。尽快送达还必须处于当日营业时间；预约日期限定为上海时区今天起 7 天内，时段必须启用，今天预约必须早于截止时间。
- 预约容量按“预约日期 + 店铺时段”统计非取消订单；达到上限时返回 `DELIVERY_SLOT_UNAVAILABLE`。预览只作提示，创建事务内必须再次统计并校验。
- 起送额、配送费与预计时效使用地址小区对应的店铺配送覆盖配置；商品总额未达到起送额时返回 `MINIMUM_ORDER_NOT_REACHED`。
- 预览返回服务端购物车版本和预计应付金额；创建必须回传该版本与金额。任一变化返回 `ORDER_PREVIEW_STALE`，客户端重新预览并要求居民确认最新结果。
- 订单状态初始为 `PENDING_PAYMENT`，支付过期时间为创建后 15 分钟；支付成功只能由可信微信通知或服务端查单写入，不得由客户端伪造、重复增加销量或提前释放库存。

## 快照与隐私

- 地址快照保存收货人、联系手机号、小区名称和完整门牌；商品快照保存商品 id、名称、图片、成交单价、数量和小计；后续地址或商品编辑不得改变历史订单。
- `users.phone` 只允许由居民点击微信原生手机号授权按钮后，以一次性动态 code 经服务端换取并绑定；不得把地址表单中的联系手机号反写为已验证账号手机号。
- 新建地址可用当前居民已验证的账号手机号预填空白联系手机号，居民仍可修改；订单始终以居民确认的地址联系电话保存履约快照，后续账号手机号或地址编辑不得改变历史订单。
- 创建请求指纹仅保存参数的 SHA-256 摘要；日志、状态说明和幂等记录不保存完整手机号、地址或用户备注。

## 状态日志

每次变化保存订单、前后状态、操作人类型（user/admin/system/wechat）、操作人标识与名称、说明和时间。状态更新和日志写入处于同一事务。

## 列表、详情与允许动作

- 居民列表和详情只读取当前登录居民的订单快照；跨居民订单统一返回 `ORDER_NOT_FOUND`。列表可按单一订单状态筛选并按创建时间倒序分页，卡片返回店铺快照、订单号、商品摘要、实付/应付金额、状态、下单时间和当前允许动作。
- 居民详情返回商品、地址、配送、金额和店铺联系快照，以及去除内部备注和操作人身份的公开状态时间线。未过期待付款订单允许取消；已支付未接单订单允许发起整单退款，存在退款申请后只展示退款进度。
- 平台管理员列表支持订单号、地址联系电话、店铺、小区、订单状态、配送方式和创建时间筛选；列表手机号默认脱敏，详情在已鉴权平台管理员范围内返回履约所需联系快照、内部备注和完整状态日志。
- 后台不提供把订单改为 `PAID` 的动作；支付成功只能由可信微信通知或服务端查单写入。后台只允许 `PAID -> ACCEPTED -> PREPARING -> WAITING_DELIVERY -> DELIVERING -> COMPLETED` 的履约动作，以及通过统一安全关单流程关闭 `PENDING_PAYMENT`。
- 居民取消、后台关闭或系统超时处理待付款订单时：没有微信交易可直接关闭；存在交易则先查单，`SUCCESS` 进入支付成功，`NOTPAY` 调用微信关单，`CLOSED` 可直接确认关闭，外部结果未知时保持待处理且不得释放库存。确认关闭后在同一事务中把状态改为 `CANCELLED`、将 `stock_released` 从 false 置为 true、恢复商品库存并写状态日志；重复请求不重复恢复库存或写日志。
- 超时任务按 `expires_at <= now` 分批读取候选，支付关闭领取具有可过期租约；多 worker、居民取消和后台关闭可安全重放，进程重启后继续处理未完成租约。
- 居民退款申请与后台接单竞争时先锁定订单：只有一个动作能从 `PAID` 成功；退款审核拒绝原子恢复 `PAID`，可信退款成功原子进入 `REFUNDED`、恢复一次库存并写状态日志。
- 状态更新先锁定订单并以当前状态条件提交。相同动作重放到已达到的目标状态视为幂等成功；其他非法或竞态跳转返回 `INVALID_ORDER_STATUS`，不修改订单、库存或日志。
- 居民订阅消息与后台提醒从已提交的状态日志和当前订单状态异步派生；提醒失败、延迟、已读或未知结果均不得反向改变订单状态。已支付订单超过 10 分钟未接单只生成后台提醒，不自动退款或释放库存。
- 订单保存支付、接单、制作、待配送、开始配送、完成与取消时间；未发生节点返回 null。后台备注最长 500 字，只向管理员返回，不进入居民时间线。

## 验收

- 并发下单不超卖；事务失败不留下半订单。
- 重复取消、关单任务或回调不重复恢复库存/销量。
- 后台和小程序读取同一订单状态真相；页面把状态映射为用户可理解的进度。
- 禁止删除订单、订单项和状态日志。
