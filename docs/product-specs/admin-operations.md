# 后台运营

来源：需求说明书第 22-31、47-51 页。

## 第一版模块

- 管理员登录与基础 RBAC。
- 看板：有效订单、成交金额、用户/店铺/商品基础指标。
- 小区、店铺、店铺配送范围、配送时间、商品分类和商品管理。
- 订单列表/详情/允许的状态修改，退款审核和结果查看。
- 用户只读列表/详情、后台管理员管理、操作日志与基础系统配置。

## 行为规则

- 管理员使用用户名和密码登录；服务端签发 8 小时 JWT，会话访问仍实时校验账号启用状态。
- 登录失败不暴露账号不存在或已禁用；同一账号连续失败 5 次后锁定 15 分钟，每次尝试写入去敏登录日志。
- 配送小区包含名称、城市、区县、详细地址、启停状态和排序值；列表支持关键字、状态筛选和分页，删除为软删除并自动停用。
- 店铺维护名称、电话、地址、营业时间、起送额、配送费、预计送达时间、营业状态和多个启用小区；金额 API 固定使用两位小数字符串，配送范围更新必须原子完成。
- 店铺可选上传一张 Logo 和一张封面图；商品可选上传一张主图及最多 6 张轮播图。后台提供选择、上传、预览和移除入口，保存后居民端店铺卡片/详情及商品列表/详情使用同一份受控图片 URL；未上传时继续使用现有文字占位，不阻断创建或上架。
- 图片上传只接受 PNG、JPEG、WebP，单张服务端限制为 512 KiB。管理员每次只上传一张；超过限制时管理后台先自动缩放并降低质量重新编码，再提交压缩结果，同时提示已压缩。浏览器无法解码的文件会提示选择有效图片，不会提交不可验证的内容。服务端仍校验最终文件的扩展名、声明 MIME、文件头和解码后的实际大小，并生成不可由客户端指定的媒体标识；压缩后仍超限或校验失败不写媒体记录，也不能把外部任意 URL 写入店铺或商品字段。
- 商品分类属于单一店铺；停用分类时其中在售商品自动下架，有未删除商品的分类不可删除。单规格商品维护两位小数价格、库存、限购、预警、热销和上下架状态；零库存、停用分类或停用店铺下不能上架。
- 店铺分别配置“尽快送达”和“预约配送”开关，至少保留一种配送方式；预约时段是每日重复的固定送达时刻，不是起止时间区间。
- 每个预约时段包含送达时间、停止下单时间、最大订单数、启停状态和排序值；停止下单时间必须早于同日送达时间，同一店铺不能重复配置相同送达时刻，最大订单数必须大于零。停用时段保留历史配置。
- 列表统一支持筛选、分页、状态展示和可恢复错误。
- 后台提供居民用户只读列表和详情。微信身份首次登录即创建居民记录；首次手机号授权成功后，同一记录立即更新为“已绑定”并出现在列表结果中，无需居民再次登录或下单。
- 用户列表展示居民主动保存的头像和昵称（未完善时使用文字头像与“微信用户”回退名）、用户 ID、脱敏手机号、手机号绑定状态、当前小区、账号状态、注册时间和最近登录时间；详情沿用同一份只读展示资料。不得返回头像二进制、OpenID、UnionID、完整手机号、地址明细或微信凭证。订单履约需要的完整联系电话仍只在已鉴权订单详情中按既有规则提供。
- 本切片的用户管理为只读查询、筛选、分页和详情，不新增物理删除、修改微信身份或后台手工绑定手机号能力。
- 提醒中心从订单状态日志、当前待接单订单、待审核退款和商品库存读取新单、10 分钟未接单、退款申请及低库存提醒；提醒只保存去敏业务摘要，不作为订单、退款或库存真相。
- 管理员可标记提醒已读；订单接单、退款审核完成或库存恢复会自动解决对应提醒。订单恢复待接单或库存再次不足时重新打开提醒。提醒音仅在管理员当前页面会话中明确开启后播放，并只用于新单和超时未接单。
- 删除店铺、商品、分类和小区是软删除；被历史订单引用的数据不丢失。
- 订单列表支持订单号、地址联系电话、店铺、小区、订单状态、配送方式和创建日期筛选；列表联系电话默认脱敏，详情才返回履约所需完整快照、状态日志和内部备注。
- 订单状态只显示状态机允许的动作，服务端仍做最终校验；后台可关闭待付款订单，已支付后只能按接单、制作、待配送、配送中、完成顺序推进，不能手工写入已支付。
- 后台关闭待付款订单时原子恢复库存一次；状态动作携带期望状态并可安全重放，非法或竞态跳转不写状态日志。内部备注最长 500 字，只向管理员返回且每次修改写操作日志。
- 退款列表支持按订单号和退款状态筛选；只有待审核申请显示通过/驳回动作。通过后由服务端复用固定商户退款单号提交/查询微信，驳回原因必填并恢复订单待接单；外部结果未知或异常不得显示成功或释放库存。
- 每个写操作记录管理员、资源、动作、变更摘要、IP/request id 和时间；敏感字段不进入 diff。
- 看板业务日固定使用 `Asia/Shanghai`：今日订单按创建时间统计且包含未支付/取消；有效支付订单按支付时间统计并排除退款成功，退款审核中仍计入有效订单。
- 今日净成交额按“今日成功支付金额 - 今日退款成功金额”计算，同时返回支付、退款和净额三项；支付/退款笔数按各自可信成功时间归日，退款审核中不提前扣减。
- 趋势只允许近 7 天或 30 天，店铺表现按同一服务端口径返回；前端不得从订单列表自行求和。待接单、制作中、配送中和退款处理中读取当前权威状态。
- 操作日志列表不返回 diff，详情才返回服务端白名单摘要；白名单只含状态、名称、排序、范围 ID、配送配置、价格、库存等业务字段，完整手机号、地址、微信身份、Token/密钥、备注和自由文本详情均不落入摘要。
- 操作日志保存独立 request id、IP、请求路径和业务标识，只提供查询、筛选、分页与详情 API，不提供修改或删除 API；历史日志查询仍执行一次去敏以防旧记录泄露。
- 店铺管理员能力默认不可用，即使数据模型已预留。

## 验收

- 管理员可以建立“小区 -> 店铺 -> 配送范围 -> 分类 -> 商品”的完整基础数据链。
- 管理员可以分别上传、预览、替换或移除店铺 Logo/封面和商品主图/轮播图；未登录上传、扩展名/MIME/文件头不一致、超限图片及超过 6 张轮播图均被拒绝，居民端只能读取受控图片内容。
- 管理员可以为店铺切换配送方式，并创建、编辑和停用每日预约配送时段。
- 居民首次绑定微信手机号后，管理员无需等待下单即可在用户列表看到该居民及脱敏手机号；未绑定居民也能以明确的“未绑定”状态查询。
- 无权限、禁用账号和跨店访问返回稳定错误，界面不泄露目标数据是否存在。
- 看板只统计已支付有效订单；退款后的统计口径在指标旁可解释。
- 看板 API 同时返回支付总额、退款成功额和净额，金额固定两位小数；日志详情无法检索出完整手机号、地址、Token 或自由文本敏感字段。
