# 后台运营

来源：需求说明书第 22-31、47-51 页。

## 第一版模块

- 管理员登录与基础 RBAC。
- 看板：有效订单、成交金额、用户/店铺/商品基础指标。
- 小区、店铺、店铺配送范围、配送时间、商品分类和商品管理。
- 订单列表/详情/允许的状态修改，退款审核和结果查看。
- 用户只读列表/详情、后台管理员管理、操作日志与基础系统配置。

## 行为规则

- 列表统一支持筛选、分页、状态展示和可恢复错误。
- 删除店铺、商品、分类和小区是软删除；被历史订单引用的数据不丢失。
- 订单状态只显示状态机允许的动作，服务端仍做最终校验。
- 每个写操作记录管理员、资源、动作、变更摘要、IP/request id 和时间；敏感字段不进入 diff。
- 店铺管理员能力默认不可用，即使数据模型已预留。

## 验收

- 管理员可以建立“小区 -> 店铺 -> 配送范围 -> 分类 -> 商品”的完整基础数据链。
- 无权限、禁用账号和跨店访问返回稳定错误，界面不泄露目标数据是否存在。
- 看板只统计已支付有效订单；退款后的统计口径在指标旁可解释。
