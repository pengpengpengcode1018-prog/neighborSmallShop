# 支付与退款

来源：需求说明书第 16-20、32-37、47-50 页。

## 支付

- 居民只能为自己的 `PENDING_PAYMENT` 订单初始化支付；服务端使用订单号、订单金额快照、账号 OpenID 和订单 `expiresAt` 创建微信 JSAPI/小程序支付单，不接收客户端金额、OpenID 或商户号。
- 支付初始化返回小程序调起所需的 `timeStamp`、`nonceStr`、`package`、`signType` 和 `paySign`；同一订单只保留一个支付记录和商户订单号，重复点击复用已有预支付单或返回明确处理中状态。
- 回调不校验居民登录态，必须对原始请求体使用 `Wechatpay-Serial`、`Wechatpay-Signature`、`Wechatpay-Timestamp` 和 `Wechatpay-Nonce` 验签；验签成功后才使用 APIv3 key 解密 `AEAD_AES_256_GCM` 资源。
- 平台验签公钥以“公钥 ID -> PEM”映射配置；响应和回调必须按 `Wechatpay-Serial` 精确选取对应公钥，不得逐把试验。轮换窗口允许旧、新公钥并存，未知 ID 或 ID/签名错配直接拒绝；服务端调用微信支付 API 时携带当前主动公钥 ID。
- 回调验签后验证通知类型、AppID、商户号、商户订单号、微信交易号、币种、支付金额和当前状态；签名时间超出允许窗口或通知 id 重放不得再次执行业务。
- 相同回调可重复通知；只允许一次把订单从待支付推进到已支付并增加销量。
- 支付成功事务同时更新支付记录、订单 `PAID/paidAt`、商品销量、通知摘要和订单状态日志；任一失败整体回滚，原始通知和密钥不落库。
- 支付失败、用户取消调起和支付结果未知是不同状态；小程序调起结果只用于界面提示，随后通过服务端状态查询恢复，不能猜测成功。
- 商户系统不能只依赖通知；待支付详情和支付调起返回后可按商户订单号查询微信，可信查询结果与通知共用同一幂等入账事务。
- 微信 `prepay_id` 最长有效 2 小时，但本产品订单仍以 15 分钟 `expiresAt` 为支付结束时间；超过订单到期时间不得新建支付单。
- 一旦支付单进入创建中或待支付，本地取消/后台关闭不得先释放库存；统一关单流程先查明未支付并关闭微信交易。支付成功通知与取消并发时以可信支付结果优先，避免出现已收款但本地订单取消。
- 微信 `time_expire` 只是支付截止时间，不等于交易已经关闭；居民取消和订单到期都要执行“查单 -> 未支付则关单 -> 本地关闭”的恢复协议。微信关单成功返回无响应体，失败或网络未知后再次查单；未确认 `CLOSED` 前不得把支付标为关闭。

## 退款

- 第一版退款等同于居民取消“已支付但商家未接单”的整单订单；仅订单本人可申请，客户端提交请求 id、预定义原因和可选说明，不提交金额、交易号或退款成功状态。
- 一单只允许一条整单全额退款记录，金额取服务端支付记录并使用 Decimal；单次及累计退款都不得超过实际支付金额。部分商品、多次部分退款和已接单后的售后不在第一版范围。
- 同一申请请求安全重放返回原退款；同一商户退款单号在微信申请失败或网络未知时必须原号重试，不得生成新号造成重复退款。
- 居民申请后退款为 `PENDING_REVIEW`，订单进入 `REFUND_PENDING` 并阻止接单；管理员可审核通过或拒绝。拒绝时退款为 `REJECTED`，订单回到 `PAID`；审核、拒绝原因和操作人写入不可删除审计。
- 审核通过先持久化 `APPROVED` 和固定商户退款单号，再离开数据库事务调用微信。微信受理后按返回映射为 `PROCESSING|SUCCESS|FAILED`；申请接口成功只表示受理，不得直接猜测退款完成。
- 微信返回 `SUCCESS` 时才把订单置为 `REFUNDED`、恢复一次库存并写完成时间；`PROCESSING` 保持待处理，`CLOSED|ABNORMAL` 记录稳定失败原因且不伪装成功。
- 商户退款单号查单与退款结果通知共用同一终态事务；校验商户号、原订单号、微信交易号、商户退款单号、退款号、币种和服务端金额。通知原文不落库，只保存外部通知 id、SHA-256 摘要、状态和幂等处理结果。
- 居民退款详情展示编号、原订单号、退款金额、原因、说明、审核说明和申请/审核/完成时间；管理员列表/详情支持待审核处理，但居民隐私仍按订单后台规则最小化展示。

## 安全与验收

- 生产支付、退款和证书操作必须显式授权；开发测试使用微信沙箱/fixture。
- 支付/退款日志脱敏，不记录密钥、完整回调敏感体或完整手机号。
- 支付通知需在 5 秒内完成验签并应答；成功响应可安全重放，验签失败返回非 2xx 并且不接触订单数据。
- 测试覆盖重复支付回调、金额/商户/AppID 不符、无效/过期签名、通知与取消竞态、查询恢复和销量只增加一次；退款测试在退款切片补充累计越界、重复请求和网络超时恢复。
