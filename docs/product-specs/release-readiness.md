# 发布就绪与门禁

来源：需求说明书第 37-38、48、51 页。

## 可自动验证的行为

- `npm run verify` 是仓库级必过门禁，并包含密钥文件、生产容器与 Compose 的静态检查。
- 生产配置拒绝默认/占位 JWT、开发数据库凭证、不完整微信配置、非 HTTPS 或示例支付回调；选择 `formal` 时必须同时具备登录、支付和订阅模板配置。
- API 只接受最长 128 字符的安全 request id；外部异常值替换为服务端 UUID。接口和登录使用独立频率上限并返回统一 `429 RATE_LIMITED`。
- `npm run release:rehearse` 使用临时随机凭证和独立数据卷，构建后台/API 镜像，执行迁移，验证 MySQL/Redis readiness、未登录后台 401、只读非 root 运行时和 API 重启恢复，结束后清理临时凭证与卷。
- API 生产镜像不包含 Prisma CLI 或前端/小程序构建链；后台镜像只包含静态产物和非 root Nginx。MySQL/Redis 只在内部网络，API 和后台分别以 loopback 端口交给宿主 Nginx；后台生产构建请求专用 API 域名，API CORS 只允许精确的后台 HTTPS Origin。
- `compose.production.yaml` 不启动 MySQL，要求通过 ECS/RDS 同 VPC内网地址连接独立 RDS 数据库；未购买 Tair 时 Redis 只在 ECS backend 网络运行，启用密码、AOF、内存上限且不发布 6379。

## 交易与安全验收矩阵

| 风险                | 自动证据                                                     |
| ------------------- | ------------------------------------------------------------ |
| 重复下单与参数变更  | 同一居民/request id 并发只建一单；同参数回放，异参数冲突     |
| 库存与时段超卖      | 条件扣减和时段行锁并发测试只有一方成功                       |
| 支付重复初始化/回调 | 单支付记录、处理中状态、通知幂等入账与销量只增加一次         |
| 取消/超时/退款竞态  | 关单租约、库存只恢复一次、审核固定退款单号与终态幂等         |
| 居民/管理员越权     | token kind 隔离、账号状态重验、资源归属和未登录 401/越权 403 |
| 隐私与审计          | 地址不持久化客户端、后台列表脱敏、日志白名单与读取二次去敏   |

## 不能由仓库自动宣告通过的条件

- 正式 AppID、微信主体、隐私保护指引、合法域名、备案、HTTPS 证书和小程序审核必须在微信/基础设施平台人工确认。
- 真实支付、退款和订阅消息必须使用批准的测试订单真机联调；fixture 只能证明代码契约，不能证明平台配置或资金链路。
- 依赖公告、备份恢复、生产监控、管理员交付和安全风险接受必须按 `docs/RELEASE_CHECKLIST.md` 关闭。自动化全绿但人工项未关闭时，生产结论仍为 `NO-GO`。
