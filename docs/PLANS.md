# 执行计划规则

## 何时必须建计划

跨会话、跨运行单元、涉及交易/安全/上线风险或存在开放决策时，必须创建 execution plan。小型局部改动可在当前计划中追加一个有边界步骤。

## 生命周期

- `exec-plans/active/`：当前驱动工作的计划；同一时刻只有一份非索引文档。
- `exec-plans/completed/`：已验证完成的历史计划，不删除。
- `exec-plans/tech-debt-tracker.md`：已确认且明确延期的债务。

## 计划最少内容

- 目标与用户可见结果
- 范围和明确非目标
- 当前唯一步骤
- 验证路径与证据位置
- 风险、blocker 和开放决策
- 带日期的进度日志

计划随执行更新。改变实现方向的决定必须写回计划；只有验证完成且证据记录后才能归档。
