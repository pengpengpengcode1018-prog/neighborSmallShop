# 微信登录与用户会话

- 状态：Completed
- 日期：2026-07-16
- 关联功能：`wechat-login`
- 前置计划：`../completed/2026-07-16-delivery-slots-admin.md`

## 目标

居民可通过微信身份进入小程序，服务端安全创建或恢复用户会话；本地开发和自动化测试在不使用真实微信凭证的情况下验证相同业务边界。

## 范围

- 核对微信登录、手机号绑定和会话恢复的原始需求，固化用户状态与稳定错误契约。
- 增加用户模型、migration、repository/service/controller/route 和去敏登录审计。
- 通过 provider/adapter 隔离微信 `code2Session`，生产适配器只从环境读取凭证，测试使用确定性假实现。
- 实现小程序登录入口、会话持久化、恢复、过期处理和可恢复错误状态。
- 覆盖首次登录、重复登录、无效 code、禁用用户、provider 失败、token 过期和敏感数据不进入日志。

## 非目标

- 不实现购物车、地址、小区选择、支付、订阅消息或真实手机号解密。
- 不把 AppSecret、session key、openid、token 或完整手机号写入代码、日志、截图和文档。
- 不在缺少用户授权和正式凭证时调用真实微信接口。

## 当前步骤

计划完成。下一功能切片由 `../active/2026-07-16-community-selection.md` 驱动。

## 完成结果

- `POST /api/v1/auth/wechat-login` 和 `GET /api/v1/users/profile` 遵循统一响应契约；首次/重复登录、禁用账号、过期 token 与管理员 token 串用均已覆盖。
- 生产微信 provider 只从环境读取凭证，5 秒超时，丢弃 session key，并把微信原始故障归一为稳定错误；测试通过注入假 provider，不存在运行态登录旁路。
- 小程序提供游客状态、微信登录、token 持久化、启动恢复、过期清除、退出与可恢复错误提示；手机号不作为首登门槛。
- migration、全量验证和构建产物冷启动通过；缺少微信凭证时登录安全返回 503，响应及结构化日志均不含临时 code。

## 验证路径

```bash
npm run db:migrate
npm run test --workspace @nearby-shop/server
npm run verify
```

真实微信能力只在用户提供有效开发凭证并明确允许外部调用后验证；默认使用本地假 provider 和小程序编译证据。

## 已确定决策与风险

- 需求说明书第 39 页将微信登录与手机号绑定列为独立接口，因此手机号不是首登硬门槛；当前功能不采集手机号。
- 游客可以浏览店铺和商品；个人资料及后续个人/交易能力使用居民 token，且管理员 token 与居民 token 严格隔离。
- 运行态不提供登录旁路；自动化测试通过注入确定性假 provider 验证相同业务边界。
- 微信 `code` 和 session key 都是短期敏感数据；日志与错误上下文必须机械去敏。
- JWT 只代表本服务会话，后续每次敏感操作仍需实时校验用户状态。
