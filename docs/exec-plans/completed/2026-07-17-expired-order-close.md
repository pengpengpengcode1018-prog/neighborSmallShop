# 超时关单与库存恢复

- 状态：Completed
- 日期：2026-07-17
- 关联功能：`expired-order-close`
- 前置计划：`2026-07-17-wechat-payment.md`

## 目标

待支付订单在 15 分钟到期后可安全关闭，居民主动取消或系统超时时都先确认并关闭未支付微信交易，库存与配送容量只恢复一次。

## 范围

- 复核需求说明书关单、超时处理和库存恢复页及微信支付官方查单/关单契约。
- 扩展微信支付 provider 的关单能力，定义支付成功、未支付、已关闭和外部结果未知的恢复语义。
- 实现居民取消与系统超时的统一关单 service/job：先锁定候选范围，逐单查明支付真相，必要时关闭微信交易，再原子关闭订单、支付记录、恢复库存并写审计。
- 定时任务支持批量上限、安全重放、并发 worker、支付通知竞态和进程重启恢复。
- 小程序对已过期/已关闭订单显示稳定状态，不能继续调起支付。

## 非目标

- 不实现退款、订阅消息、后台告警或生产任务调度平台。
- 不执行真实商户关单；本地使用 provider fixture，真实验证仍需显式批准与商户凭证。

- `20260717170000_expired_order_close` migration 增加支付 `CLOSING` 状态、关单原因/操作人、领取/最近尝试/次数和完成时间，支持租约恢复与去敏审计。
- 微信支付 provider 增加商户订单查单身份校验和签名关单；居民取消、管理员关闭、系统超时统一走三阶段关单 service，外部结果未知不释放库存。
- 超时 job 启动即运行并按间隔/批量配置扫描，进程内防重叠，数据库关闭租约支持多 worker、支付通知竞态和进程重启恢复。
- 支付成功查询与通知共用幂等入账事务；金额不符保留订单与库存，确认关闭才原子取消、恢复一次库存并写状态/管理员审计。
- 小程序订单列表、详情和创建成功页识别过期状态，过期后隐藏或禁用支付并展示正在确认关闭。

## 风险与开放决策

- 外部查单或关单网络结果未知时不能先释放库存；需保留可重试状态，避免交易随后成功却已取消订单。
- 微信支付成功通知与关单并发时，订单行锁和外部交易真相必须保证只有正确终态生效。
- worker 不能长时间持有数据库事务等待微信网络；候选领取、外部调用和最终提交需要可恢复的分阶段协议。
