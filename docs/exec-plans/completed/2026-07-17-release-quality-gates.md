# 发布质量门禁

- 状态：Completed
- 日期：2026-07-17
- 关联功能：`release-quality-gates`
- 前置计划：`2026-07-17-operations-dashboard-audit.md`

## 目标

把第一版已有功能收束为可重复的发布门禁：验证并发与幂等不变量、鉴权和隐私边界、依赖与密钥配置，以及 Docker 冷启动和重启恢复，使“可以本地运行”提升为“具备明确上线条件”。

## 范围

- 可视复核需求说明书第 37-38、48、51 页，建立并发、安全、隐私、依赖、HTTPS/微信配置和部署验收矩阵。
- 盘点现有交易、越权、去敏、回调、任务恢复和架构检查覆盖，缺口优先升级为自动测试或 `scripts/` 门禁。
- 验证 Docker Compose 构建、干净启动、迁移、readiness、进程/容器重启和关键黄金旅程；记录真实微信平台仍需人工联调的门禁。
- 复审生产环境变量、日志/截图/测试数据和前端构建产物，不提交真实密钥或居民敏感数据。
- 更新发布清单、质量评分、可靠性、安全和技术债，执行最终全量验证。

## 非目标

- 未实际部署生产、购买域名/证书/云资源，未执行真实支付、退款或订阅消息发送。
- 未在缺少真实平台凭证时伪造“微信联调通过”；fixture 证据与生产门禁分开记录。
- 未扩张商家后台、营销、结算、配送员或第二阶段 RBAC。

## 完成结果

- `docs/product-specs/release-readiness.md` 与 `docs/RELEASE_CHECKLIST.md` 区分自动化通过和生产人工 `NO-GO`；正式微信、HTTPS、隐私审核、监控备份和构建链 high 都有明确负责人输入。
- 生产配置拒绝占位 JWT、开发数据库、无认证 Redis、不完整微信配置、示例/非 HTTPS 回调和缺少模板的 `formal` 状态；API 新增有界限频、request id 清洗和 JSON-only/1 MiB 请求体。
- API 与后台使用独立锁且在线审计均为 0 漏洞；API 生产镜像不含 Prisma CLI，后台运行镜像只含非 root Nginx 与静态产物。
- `compose.release.yaml` 提供内部 MySQL/Redis/API 网络、一次性迁移、只读非 root 运行时与后台入口；`release:rehearse` 使用随机临时凭证和空卷并默认清理。
- 根锁仍有 0 critical、9 high、14 moderate、11 low，因此门禁功能已完成但生产发布结论保持 `NO-GO`。

## 验证证据

- 需求说明书第 37-38、48、51 页经 PDF 渲染后可视复核。
- `npm run release:audit:runtime`：API 独立生产锁 0 漏洞；后台独立构建锁安装审计 0 漏洞。

- `npm run verify`：文档、架构、两份 Compose、发布安全、lint、类型、130 项服务端测试、44 项小程序测试、三端构建和格式全部通过。

## 风险与延期工作

- 根构建链 9 high 未清零或书面接受前不得公网/生产发布；API/后台隔离锁为 0 不能替代整体供应链判断。
- 正式微信主体/AppID/商户、HTTPS 合法域名、隐私审核、真机资金链路、监控备份与恢复均需外部平台和负责人输入。
- 当前限频与定时任务面向单 API 容器；多副本前必须升级为共享限频、独立调度、积压指标和告警。
