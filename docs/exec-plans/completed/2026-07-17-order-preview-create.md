# 订单预览、创建与库存锁定

- 状态：Completed
- 日期：2026-07-17
- 关联功能：`order-preview-create`
- 前置计划：`../completed/2026-07-16-address-management.md`

## 目标

登录居民可基于当前服务端购物车、有效地址和配送方式获得可信订单预览，并以幂等请求原子创建一个待支付订单和锁定库存。

## 范围

- 核对需求说明书第 14-16、34-36、40、47-48、50 页，固化预览字段、地址/配送校验、金额快照、订单号、库存条件更新和重复请求规则。
- 增加订单、订单商品、地址/价格快照、创建请求幂等键和必要状态日志 migration。
- 提供订单预览与创建 API；服务端重算商品、费用、起送额、地址范围和配送时段，不信任客户端金额或库存。
- 创建订单在单个事务中条件扣减库存、保存快照并清空购物车；同一居民重复请求只返回同一订单。
- 实现小程序订单确认页及提交过程的 loading/error/retry/金额变化确认状态。

## 非目标

- 不接入真实微信支付、支付回调、超时关单、库存恢复、订单列表或后台状态流转。
- 不允许跨店购物车、多规格商品或客户端直接决定订单金额。

- 创建事务按居民与预约时段加行锁，条件扣减库存，写订单/商品/首条状态日志，更新地址最近使用并清空购物车；任一失败整体回滚。
- 小程序完成地址选择、即时/预约配送、备注、服务端金额、最新内容确认、网络失败同请求重试与待支付成功态，不伪造支付。

## 风险与开放决策

- 库存条件更新、订单写入、购物车清空和幂等记录必须处于同一事务；并发超卖与重放先写失败测试再实现。
- 订单必须保存不可变商品、金额、地址和配送快照，后续主数据编辑不能改变历史订单。
- 支付尚未接入时订单只到 `PENDING_PAYMENT`；不得伪造支付成功或提前增加销量。
- 需求书要求下单前绑定手机号，但当前没有可信微信手机号授权流程；本切片只把居民明确填写的地址联系电话保存为订单快照，不写入 `users.phone` 或宣称已验证，正式授权仍是发布前门禁。
- 预约日期在缺少商家营业日历的第一版限定为上海时区今天起 7 天内；今天预约必须早于时段截止时间，未来功能再扩展停业日配置。
