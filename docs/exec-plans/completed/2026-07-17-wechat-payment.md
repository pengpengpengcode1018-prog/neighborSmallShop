# 微信支付与幂等回调

- 状态：Completed
- 日期：2026-07-17
- 关联功能：`wechat-payment`
- 前置计划：`2026-07-17-order-status-flow.md`

## 目标

居民可从待付款订单安全获取微信支付调起参数，可信微信通知只能一次把订单推进为已支付并增加销量，失败、取消和未知结果都有可恢复语义。

## 范围

- 核对需求说明书第 16-17、35-37、40、47-50 页和微信支付官方契约，固化支付单、调起参数、通知验签和结果查询边界。
- 增加支付记录、微信交易标识、通知幂等键、金额和审计字段 migration；订单、支付与销量更新保持单事务一致性。
- 以 provider 契约隔离微信下单、通知验签和交易查询；测试默认使用本地 fixture，不记录私钥、证书或敏感通知原文。
- 提供待付款订单支付初始化与微信通知 API；验证商户、订单号、金额、当前状态和重复通知。
- 小程序接入 `uni.requestPayment`，区分用户取消、明确失败、网络未知和服务端已确认支付，并在恢复后刷新订单真相。

## 非目标

- 不执行真实生产支付、不配置或提交商户私钥/平台公钥；真实凭证和外部交易必须另行显式批准。
- 不交付退款、15 分钟定时关单、订阅消息或营销优惠。

## 完成结果

- `20260717160000_wechat_payments` migration 增加每单唯一支付记录、商户订单号、微信交易号及去原文通知/查单审计。
- 官方 V3 provider 完成 JSAPI 下单请求签名、响应验签、小程序调起签名、回调原始体验签/AES-256-GCM 解密和商户订单查单；配置只从环境读取。
- 支付 API 只接收订单 ID，金额、订单号、OpenID 和到期时间均取服务端；重复初始化复用预支付单，并发创建返回稳定处理中状态。
- 可信通知或查单在订单行锁下原子更新支付、订单、销量、通知摘要和状态日志；相同及不同通知重放不重复入账。
- 活跃微信支付会阻止本地先取消和恢复库存，避免已收款但订单已取消；关闭微信交易后取消由下一关单切片交付。
- 小程序订单创建页和详情页可调起微信支付；客户端结果后再次查询服务端，分别呈现已确认、用户取消、明确失败或未知状态。

## 风险与开放决策

- 本地没有可提交的微信支付商户私钥与平台公钥；自动化使用运行时生成的 RSA/AES fixture，不能宣称真实沙箱或生产支付已联通。
- 原计划交付时只接纳一把平台公钥；2026-07-20 的发布收口切片已改为按 `Wechatpay-Serial` 精确选择多把公钥并补齐轮换窗口自动化。真实商户灰度轮换演练仍是发布门禁。
- 微信支付创建中/待支付时暂不允许居民取消或后台关闭；下一切片需先查单并关闭微信交易，再执行订单取消、超时关单和库存恢复。
- 微信开发者工具自动化仍未启用；小程序以 store 测试和 `mp-weixin` 构建作为本地证据，正式 AppID/商户号/真机支付是发布门禁。
