# 居民端商品分类、列表与详情

- 状态：Completed
- 日期：2026-07-16
- 关联功能：`product-browsing`
- 前置计划：`../completed/2026-07-16-store-browsing.md`

## 目标

居民进入可见店铺后，可按分类浏览公开商品、理解在售/售罄状态并查看商品详情；商品资格与库存状态完全由服务端给出。

## 范围

- 核对需求说明书第 10-12、39、42、47、49 页，固化分类、在售/售罄/下架、库存和详情字段规则。
- 提供居民端店铺分类与商品列表、商品详情 API，复用店铺与当前小区的可见性边界。
- 只返回启用、未删除分类下的非下架商品；售罄商品可展示但不可购买，下架或软删除商品隐藏。
- 实现店铺页分类和商品 loading/empty/error/retry、售罄状态及商品详情基础信息。
- 覆盖跨店分类、无效/不可配送店铺、零库存、售罄、下架、软删除、价格字符串和公开字段。

## 非目标

- 不实现加购物车、跨店清空确认、商品规格、结算或库存扣减。
- 不让客户端计算价格、库存资格或绕过店铺配送范围。

## 当前步骤

计划完成。下一功能切片由 `../active/2026-07-16-single-store-cart.md` 驱动。

## 完成结果

- 公共 `GET /api/v1/stores/:storeId/products` 返回可见分类、分页商品和服务端购买资格；`GET /api/v1/products/:productId` 提供相同边界下的公开详情。
- 停用/删除分类、下架/删除商品保持隐藏；后台售罄或实时零库存统一显示售罄，商品购买资格同时受店铺营业与当前小区配送关系约束。
- 小程序店铺页提供左侧分类、右侧商品列表、loading/empty/error/retry 和分类竞态保护；商品详情提供轮播、价格、销量、库存、正文与售后说明。
- 全量验证、`mp-weixin` 生产构建和 API 构建产物冷启动均通过。

## 风险与开放决策

- `SOLD_OUT` 状态与库存为零统一映射居民端售罄，均不可购买；`canPurchase` 还同时依赖当前店铺在所选小区可下单。
- 分类被停用或删除后，其商品不得从居民接口旁路出现；商品详情复用相同可见性判断。
- 居民商品价格使用两位小数字符串；原价为空时返回 `null`，不由客户端猜测折扣。
