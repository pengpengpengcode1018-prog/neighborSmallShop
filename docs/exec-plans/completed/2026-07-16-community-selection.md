# 小区选择与当前小区

- 状态：Completed
- 日期：2026-07-16
- 关联功能：`community-selection`
- 前置计划：`../completed/2026-07-16-wechat-login.md`

## 目标

居民可查看启用的配送小区、手动选择并在后续访问中恢复当前小区；游客可先本地选择，登录居民的选择由服务端保存。

## 范围

- 核对需求说明书第 6、8、9、39、47、49、52 页，固化游客选择、登录后同步与失效小区的处理规则。
- 提供居民端启用小区查询与受保护的当前小区更新 API，不复用后台管理响应。
- 为用户增加可空的当前小区关系；服务端只接受存在且启用、未删除的小区。
- 实现小程序选择入口、列表 loading/empty/error、选择确认、本地持久化与登录后同步。
- 覆盖禁用/删除小区不可选、无效标识、游客恢复、登录同步和已选小区后续失效。

## 非目标

- 不实现定位、地图、按距离排序、地址联动、店铺筛选、购物车或自动切换小区。
- 不让居民端看到后台备注、完整地址以外的管理字段或已停用/删除记录。

## 当前步骤

计划完成。下一功能切片由 `../active/2026-07-16-store-browsing.md` 驱动。

## 完成结果

- `GET /api/v1/communities` 公开返回启用、未删除且仅含居民所需字段的小区；`PUT /api/v1/users/current-community` 校验居民会话和有效小区。
- 用户当前小区关系已落库；已选小区后续停用或软删除时，资料恢复返回空选择并清除数据库引用。
- 小程序提供选择/切换页面、loading/empty/error/retry、游客本地持久化和登录账号同步；账号有效服务端选择优先，网络失败保留原选择。
- migration、全量验证、`mp-weixin` 构建和 API 构建产物冷启动均通过。

## 已确定决策与风险

- 游客本地选择与账号服务端选择冲突时，服务端选择优先；服务端为空时才同步有效的游客选择，避免静默覆盖跨设备状态。
- 登录状态切换必须服务端成功后再更新本地；游客选择直接本地保存。网络失败均保留原选择。
- 管理员可在用户已选择后停用或删除小区；恢复会话时必须清除或提示，不得继续作为有效配送范围。
- 当前小区是后续店铺可见性的输入，但本计划不提前实现店铺筛选。
