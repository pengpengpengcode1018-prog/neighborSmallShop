# 订阅消息与后台提醒

- 状态：Completed
- 日期：2026-07-17
- 关联功能：`notifications-alerts`
- 前置计划：`2026-07-17-refund-flow.md`

## 目标

在不伪造订单状态、不泄露居民信息且外部发送可解释恢复的前提下，为居民提供关键订单订阅消息，为平台后台提供新单、待接单超时、退款申请和库存预警提醒。

## 已完成范围

- 可视复核需求说明书第 2、4、36-37、50 页，并核对微信小程序 `wx.requestSubscribeMessage` 与服务端发送、错误码官方契约。
- 居民只通过显式点击触发一次性授权；服务端记录模板结果、请求摘要和保守可用次数，不假定永久许可。
- 订单状态日志按稳定 ID 发现消息 outbox；发送前原子领取事件并预占授权，重复扫描与并发 worker 不重复消费。
- 微信 provider 实现 access token 缓存/刷新、官方发送结构和稳定错误分类；明确未发送才重试，网络结果未知进入 `UNKNOWN` 且不自动重试。
- 后台从真实订单、退款和商品状态生成新单、10 分钟未接单、退款申请及低库存提醒，支持未读、已读、自动解决与重新打开。
- 小程序增加通知设置页和订单详情入口；管理后台增加提醒中心、未读徽标和本次会话显式开启的新单/超时提醒音。
- 新增 `notifications-and-alerts.md`，同步架构、可靠性、安全、数据库索引和技术债。

## 明确非目标

- 未执行真实生产消息发送，未提交模板 ID、AppSecret 或真实 OpenID。
- 未实现短信、邮件、营销群发、复杂消息中心、生产级队列、人工补偿台或可达率监控。
- 10 分钟未接单只提醒，不自动退款、释放库存或改变订单状态。

## 关键决策

- 微信是一次性授权最终真相；客户端报告只用于在调用前保守预占，`43101` 会清零报告次数。
- 订单状态日志是居民消息事件源，提醒与 outbox 都是派生状态，任何失败或延迟不得反向修改订单、退款或库存。
- 微信请求已经发出但网络结果未知时无法证明未投递，因此使用 at-most-once 边界：记录 `UNKNOWN`，不自动重试或归还预占次数。
- 提醒内容只保存订单号、店铺、商品与库存等必要摘要；完整手机号、姓名、门牌、备注、OpenID、token 和微信原始响应不进入提醒、消息数据或结构化日志。
- 进程遗留 `SENDING` 超过 5 分钟直接转 `UNKNOWN`；真实模板、多副本调度、积压指标和补偿流程留作发布门禁与技术债。
