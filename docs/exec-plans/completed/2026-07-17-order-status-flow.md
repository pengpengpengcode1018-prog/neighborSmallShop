# 订单列表、详情与状态机

- 状态：Completed
- 日期：2026-07-17
- 关联功能：`order-status-flow`
- 前置计划：`2026-07-17-order-preview-create.md`

## 目标

居民和管理员读取同一订单状态真相，并且订单只允许按规定路径进行可审计、幂等的接单与配送状态流转。

## 范围

- 核对需求说明书第 17-19、27-28、32-33、47-48、50 页，固化居民/后台列表详情字段、查询范围、状态文案和允许动作。
- 提供居民订单列表与详情 API，只返回当前居民订单和不可变快照。
- 提供平台管理员订单列表、详情与状态更新 API；状态变化和日志写入同一事务。
- 实现小程序订单列表/详情与后台订单列表/详情/状态操作，覆盖 loading/empty/error/retry 和并发刷新。
- 以表驱动和并发集成测试覆盖合法跳转、非法跳转、重复请求、居民范围隔离与审计日志。

## 非目标

- 不接入真实微信支付、已支付退款、15 分钟定时关单或订阅消息；待付款订单由居民取消或后台关闭时必须恢复库存。
- 不允许管理员伪造 `PAID`；支付成功只能由后续可信微信回调产生。

## 风险与开放决策

- 当前订单只能由创建流程进入 `PENDING_PAYMENT`；生产 API 不允许后台绕过支付进入 `PAID`，履约状态机使用可信已支付 fixture 验证。
- 真实微信支付、15 分钟系统关单和已支付退款仍由后续独立计划交付。
- 微信开发者工具自动化仍未启用；小程序以 store 测试和 `mp-weixin` 构建作为本地证据，正式 AppID/真机验证仍是发布门禁。
- 本轮内置浏览器与本地开发服务存在网络隔离，订单后台页面未获得新的浏览器截图证据；类型检查、生产构建、API 集成测试和既有后台布局浏览器基线均通过，该视觉复核缺口保留在质量评分中。
