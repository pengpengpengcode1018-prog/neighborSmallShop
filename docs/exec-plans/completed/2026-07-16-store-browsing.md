# 居民端店铺浏览与搜索

- 状态：Completed
- 日期：2026-07-16
- 关联功能：`store-browsing`
- 前置计划：`../completed/2026-07-16-community-selection.md`

## 目标

居民可在小程序查看可见店铺、按名称搜索并理解营业/暂停状态；选择小区后只展示支持该小区配送的店铺，并可进入安全的店铺详情。

## 范围

- 核对需求说明书第 9、10、39、40、47、49 页，固化未选小区、营业/暂停、配送范围和搜索规则。
- 提供居民端店铺列表、搜索与详情 API，只返回启用关联和非删除店铺的公开字段。
- 未选择小区时允许浏览平台可见店铺；选择小区后按有效 `store_communities` 过滤，并返回该小区覆盖后的起送额、配送费与预计时效。
- 实现首页搜索、店铺 loading/empty/error/retry、状态和费用展示，以及店铺详情基础信息页。
- 覆盖禁用/删除店铺、暂停营业、小区无配送关系、暂停配送关系、金额字符串和无效小区。

## 非目标

- 不实现商品分类/列表/详情、购物车、结算、公告轮播或基于距离的排序。
- 不让客户端自行判断配送资格或计算覆盖后的费用。

## 当前步骤

计划完成。下一功能切片由 `../active/2026-07-16-product-browsing.md` 驱动。

## 完成结果

- 公共 `GET /api/v1/stores` 支持分页、店名搜索和可选小区过滤；`GET /api/v1/stores/:id` 提供同一可见性边界下的店铺详情。
- 无小区仅提供浏览，选定有效小区后由服务端过滤有效配送关系并合并起送额、配送费和预计时效覆盖；暂停店铺保留展示但禁止下单。
- 小程序首页提供搜索、下拉刷新、loading/empty/error/retry、店铺状态与费用卡片；详情页提供公告、营业时间、联系信息及不可配送/暂停说明。
- 全量验证、`mp-weixin` 生产构建和 API 构建产物冷启动均通过。

## 验证路径

```bash
npm run test --workspace @nearby-shop/server
npm run test --workspace @nearby-shop/miniapp
npm run verify
```

小程序可见流程使用 store 自动化与 `mp-weixin` 构建；微信开发者工具服务端口只有在用户明确允许修改该持久安全设置后才启用。

## 风险与开放决策

- “首次可浏览全部营业店铺”优先用于无小区浏览；未选小区只提供浏览，不暗示可配送或可下单。
- 需求书明确暂停接单店铺继续展示，因此 `PAUSED` 店铺保留在列表和详情并禁用下单；禁用和软删除店铺始终隐藏。
- 店铺默认费用和小区覆盖费用必须由服务端合并，金额继续使用两位小数字符串。
