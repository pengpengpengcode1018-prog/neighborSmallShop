# 第一阶段管理基础能力

- 状态：Completed
- 日期：2026-07-15 至 2026-07-16
- 关联功能：`admin-auth`、`community-admin`、`store-admin`、`catalog-admin`

## 目标

交付第一条可验证的后台黄金旅程：管理员登录后，可建立小区、店铺、配送范围、分类和单规格商品，并留下授权与操作审计证据。

## 完成范围

- 应用 `20260716042000_init` migration，并提供要求显式密码、bcrypt 哈希和可重复 upsert 的开发管理员种子。
- 管理员登录、8 小时 JWT、启停实时校验、最小 RBAC、五次失败锁定和登录审计。
- 小区、店铺、配送范围、分类和单规格商品的分页 CRUD、软删除、状态约束与事务内操作审计。
- 后台登录与四个业务模块页面，以及登录、小区、店铺、分类和商品浏览器黄金旅程。
- 独立 liveness 与 MySQL/Redis readiness、构建产物冷启动和数据库连接优雅释放。

## 关键决策

- 开发种子不内置默认管理员密码；每次浏览器验证后将测试密码随机轮换。
- 金额 API 始终使用两位小数字符串；店铺配送范围在单个事务内验证并替换。
- 商品分类属于单一店铺；停用分类自动下架在售商品，零库存或无效上下文不能上架。
- `/api/v1/health` 不触碰外部依赖；`/api/v1/ready` 分别探测 MySQL 与 Redis，任一失败即返回 HTTP 503。
- Prisma 生成代码使用 `.js` 导入扩展名，使 TypeScript 编译后的 ESM 可由 Node 直接冷启动。

## 验证证据

- `npm run verify`：文档、架构、Compose、lint、类型、测试、三端生产构建和格式检查全部通过。
- 服务端 5 个测试文件共 18 个测试通过，其中认证 4 个、小区 3 个、店铺 3 个、分类/商品 4 个、健康与基线 4 个。
- 浏览器完成未登录守卫、登录/退出、小区创建编辑停用、店铺创建编辑暂停、分类与商品创建编辑旅程。
- 审计抽查：`operation_logs` 覆盖小区、店铺、分类和商品写操作，`admin_login_logs` 同时存在成功与失败记录；写日志携带管理员、目标、IP、路径和 request id，不记录密码或令牌。
- 构建产物在 3100 端口冷启动，`/api/v1/health` 与 `/api/v1/ready` 均返回 HTTP 200，MySQL 和 Redis 状态均为 `ready`；停止后端口释放。

## 遗留风险

- 密码重置、完整 RBAC 和生产凭证发放必须在生产启用前单独设计。
- 居民微信登录、小程序购物、订单、支付和退款不在本计划范围。
- DCloud/Prisma 传递依赖告警、Sass legacy API 和后台首包体积仍按技术债跟踪器处理。
