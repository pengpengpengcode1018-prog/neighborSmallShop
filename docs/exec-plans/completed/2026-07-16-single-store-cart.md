# 居民端单店铺购物车

- 状态：Completed
- 日期：2026-07-16
- 关联功能：`single-store-cart`
- 前置计划：`2026-07-16-product-browsing.md`

## 目标

登录居民可把当前可购买商品加入服务端购物车、调整数量并看到可信金额；加入另一店铺商品前必须明确确认清空原购物车。

## 范围

- 核对需求说明书第 12、13、40、47、49、52 页，固化登录、单店约束、数量、库存/限购和跨店确认规则。
- 增加居民购物车数据模型和 migration，提供查询、加入、修改、删除单项及清空 API。
- 服务端每次写入重新校验商品、店铺、小区配送、库存和限购；金额使用实时商品价格的两位小数字符串汇总。
- 实现商品列表/详情加入入口、底部购物车栏、购物车页 loading/empty/error/retry、数量调整和跨店确认。
- 覆盖未登录、跨用户隔离、单店约束、重复加入、库存/限购、失效商品、并发更新、金额与网络失败。

## 非目标

- 不实现收货地址、配送时段、订单预览、库存扣减或真实结算跳转。
- 不把客户端缓存价格、库存或购物车合计作为服务端依据。

## 当前步骤

计划完成。下一功能切片由 `../active/2026-07-16-address-management.md` 驱动。

## 完成结果

- `20260716194500_cart` migration 建立每位居民唯一购物车和购物车商品唯一约束；删除居民或购物车时安全级联清理。
- `/api/v1/cart` 提供查询、加购、改量、删除单项和幂等清空；服务端重验当前小区、配送、营业、上下架、库存与限购，并实时返回金额、起送差额和结算资格。
- 跨店加购默认返回 `CART_STORE_CONFLICT`；小程序二次确认后才发送 `replaceExistingCart: true`，服务端在事务中原子替换。
- 后来下架、售罄、超库存或超限购商品在读取时保留并附稳定失效原因；不同居民无法读取、修改或删除彼此购物车项。
- 小程序商品列表和详情提供加购入口，`CartBar` 展示服务端金额，购物车页覆盖加载、空态、失败重试、增减、失效移除和危险清空确认；退出登录清空本地购物车状态。


小程序可见流程使用 store 测试与 `mp-weixin` 正式构建验证；微信开发者工具服务端口只有在用户明确允许修改该持久安全设置后才启用。

## 风险与开放决策

- 购物车写入不锁定库存，结算前仍需重新校验；响应金额是当前展示而非下单承诺。
- 购物车的“去结算”当前只展示资格和下一功能提示；地址与订单预览完成后再接真实跳转。
- 微信开发者工具自动化未启用，因此本切片没有真机或开发者工具截图；正式 AppID 联调仍是发布前门禁。
