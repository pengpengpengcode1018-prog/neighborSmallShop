# 第一阶段管理基础能力

- 状态：Ready
- 日期：2026-07-15
- 关联功能：`admin-auth`、`community-admin`、`store-admin`、`catalog-admin`
- 前置计划：`../completed/2026-07-15-project-initialization.md`

## 目标

交付第一条可验证的后台黄金旅程：管理员登录后，可建立小区、店铺、配送范围、分类和单规格商品，并留下授权与操作审计证据。

## 范围

- 生成并应用第一份 Prisma migration，准备可重复的开发种子数据。
- 管理员登录、JWT 会话、启停校验、最小 RBAC 和登录日志。
- 小区、店铺、店铺配送范围、商品分类和商品的分页 CRUD、软删除与状态约束。
- 后台登录、列表和表单页面；API 集成测试与关键后台旅程验证。
- 数据库与 Redis readiness、错误日志和操作审计。

## 非目标

- 不实现居民微信登录、小程序购物链路、订单、支付或退款。
- 不启用店铺管理员角色；只保留未来资源范围授权的边界。
- 不使用真实微信凭证或生产数据。

## 当前步骤

在本地 MySQL 启动后创建首个 migration 和开发管理员种子；随后实现 `admin-auth`，未产生验证证据前功能状态保持 `not_started`。

## 验证路径

```bash
npm run infra:up
npm run verify
```

每个功能至少包含 API 成功、校验失败、未授权、越权、软删除和重复请求测试；用户可见后台旅程使用浏览器验证。

## 风险与开放决策

- 密码初始发放与重置流程需在进入生产前单独设计；开发种子不得复用生产凭证。
- 店铺营业状态中的 `RESTING` 由营业时间动态计算，不落库为可漂移状态。
- 跨店、跨小区关联写入必须在服务端事务内验证，不能依赖前端下拉框限制。
