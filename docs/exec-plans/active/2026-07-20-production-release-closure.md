# 正式发布门禁收口

- 状态：Active
- 日期：2026-07-20
- 关联功能：`wechat-payment`、`notifications-alerts`、`release-quality-gates`、`wechat-phone-user-admin`、`resident-profile-display`、`catalog-image-upload`
- 前置计划：`../completed/2026-07-18-wechat-live-integration.md`

## 目标

在不泄露商户和平台凭证、不伪造资金结果的前提下，逐项关闭真实支付/退款、订阅消息、微信审核、备份恢复、监控告警和依赖风险门禁；只有全部人工证据通过后才把正式发布结论从 `NO-GO` 改为 `GO`。

## 用户可见结果

- 居民在正式体验版完成批准的小额微信支付、订单履约和退款，双端状态及资金结果一致。
- 居民可在“我的”页登录并主动完善昵称和头像，与后台居民列表展示一致；跳过资料完善不影响现有能力。
- 管理员可为店铺上传 Logo/封面、为商品上传主图/轮播图，居民端读取服务端受控图片 URL；未上传时保持现有占位。
- 微信隐私保护指引、订阅消息模板、服务器域名和支付回调配置与实际功能一致。
- RDS/Redis 可恢复，关键任务和回调失败有可触达告警；正式审核前存在可执行的发布与回滚证据。

## 范围

- 以密钥系统注入商户号、APIv3 密钥、商户私钥和微信支付平台公钥映射，不在仓库、聊天、日志或截图中暴露凭证。
- 以已完成的按平台公钥 ID 精确选钥和轮换窗口自动化为基础，完成批准的真实轮换演练、真机小额支付、退款和回调幂等对账。
- 配置并验证订阅消息模板、隐私保护指引、主体/类目和审核材料。
- 完成 RDS 自动备份与恢复演练、Redis AOF 恢复、容器/磁盘/任务积压/回调失败告警。
- 清零根构建链 high 公告，或形成有负责人和到期日的书面风险接受。

## 非目标

- 不使用开发 fixture、后台改库或伪造回调替代真实资金链路。
- 不在任一人工门禁未通过时提交正式审核、开放真实居民流量或宣称已上线。
- 不通过聊天接收 AppSecret、APIv3 密钥、私钥、完整个人信息或其他生产秘密。

## 当前唯一步骤

将 `0.1.5` 候选部署到 ECS：先执行新增迁移，再仅更新 API 与管理后台容器；随后上传 `0.1.5` 小程序体验版，真机验证图片上传/展示及资料可跳过、主动保存、重新进入恢复且后台一致展示。微信隐私保护指引审核通过前不得提交正式版本审核。

## 验证路径与证据

- `npm run verify`、`npm run verify:release`、隔离运行时审计和发布演练保持通过。
- 使用批准的最小金额订单记录商户单号、支付单号、退款单号和去敏 request id；小程序、后台、RDS 与微信商户平台终态一致。
- 重复支付/退款通知不重复入账，未知外部结果只查询不盲目重试。
- RDS 恢复实例、Redis AOF 恢复及告警触达均留下不含秘密和个人信息的时间戳证据。
- `docs/RELEASE_CHECKLIST.md` 不再存在未接受的 `NO-GO` 后，才允许提交正式审核。

## 风险与开放决策

- 微信支付生产凭证已通过不泄密配置和密码学预检，批准的真实支付、退款审核、微信原路退款通知及双端终态同步均已通过；真实平台公钥灰度轮换和订阅消息模板仍未验证，因此发布继续为 `NO-GO`。
- 微信手机号显式授权、地址自动预填和后台居民列表的代码、接口、隐私测试、三端构建及正式体验版真机闭环均已通过；该功能的人工验收已关闭，微信隐私保护指引仍是全局 `NO-GO` 门禁。
- 支付代码已支持 1-8 把平台公钥并按 `Wechatpay-Serial` 精确选钥，未知 ID 与 ID/签名错配测试通过；真实商户接入后仍必须完成不暴露 PEM 的旧/新公钥灰度轮换演练。
- RDS/Redis 已通过生产 readiness，但备份恢复、任务积压和回调失败告警尚无生产证据。
- 根构建链仍有 high 公告；运行镜像隔离不能替代正式发布前的清零或书面风险接受。
