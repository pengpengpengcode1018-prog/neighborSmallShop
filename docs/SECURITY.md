# 安全规则

## 凭证与个人信息

- 只提交 `.env.example`；真实 `.env`、微信 AppSecret、支付私钥、证书和 JWT secret 不入库。
- 本地示例凭证只用于 Docker 开发容器，不得用于共享或生产环境。
- 日志禁止记录 token、完整手机号、支付密钥、证书内容、openid/session key 和微信回调敏感原文。
- API 返回个人信息遵循最小必要；后台敏感字段默认脱敏。

## 边界验证

- 请求体、路径、查询参数和微信回调全部在边界使用 Zod 校验。
- JWT 只证明身份；每个后台动作仍需角色与资源范围授权。
- 店铺管理员结构可以预留，但第一版不能获得跨店访问。
- 上传校验 MIME、扩展名、大小和对象键；文件不作为可执行内容由 Koa 直接托管。

## 交易安全

- 金额由服务端从商品快照和配送规则重新计算，使用十进制定点数。
- 创建订单使用唯一请求 id；库存条件更新和订单写入处于同一事务。
- 微信支付/退款回调先验签，再验商户、订单、金额和当前状态，最后幂等提交。
- 订单、支付、退款、状态日志和操作日志禁止物理删除。

## 外部与破坏性动作

- 生产部署、真实支付/退款、发送订阅消息、清空数据库和轮换密钥必须显式批准。
- 测试默认使用 sandbox、fixture 或本地容器；不把真实用户数据复制到开发环境。
- 新依赖在 active plan 说明用途；安全相关变更必须带滥用或越权验证。

## 依赖审计基线

- 2026-07-15 的 `npm audit --omit=dev` 结果为 0 critical、9 high、14 moderate、11 low；主要来源是官方 DCloud 固定编译批次及 Prisma CLI 传递依赖。
- 不运行会跨大版本降级或破坏 uni-app 精确 peer 的 `npm audit fix --force`；残余项记录在技术债跟踪中，并在上游兼容版本发布或任何公网/生产发布前复审。
- DCloud 开发服务器只绑定本机、只处理受信任源码和素材；生产容器不安装或运行小程序编译器与 Prisma CLI。
