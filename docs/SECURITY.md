# 安全规则

## 凭证与个人信息

- 只提交 `.env.example`；真实 `.env`、微信 AppSecret、支付私钥、证书和 JWT secret 不入库。
- 本地示例凭证只用于 Docker 开发容器，不得用于共享或生产环境。
- 生产应用仅使用 RDS 内网连接域名、ECS 私网白名单/安全组和独立最小权限账号；一次性迁移容器只接收 DDL+DML 迁移账号，API 只接收 DML 应用账号，不得把迁移凭证传给常驻运行容器。不得把 `3306` 向公网开放。受控本地迁移例外必须使用独立 DDL+DML 账号、当前公网单 IP 白名单和 CA 严格校验的 SSL，完成后替换失效 IP，禁止 `0.0.0.0/0`。ECS 内 Redis 不发布宿主端口，密码不得与 RDS/JWT/管理员凭证复用。
- 日志禁止记录 token、完整手机号、支付密钥、证书内容、openid/session key 和微信回调敏感原文。
- API 返回个人信息遵循最小必要；后台敏感字段默认脱敏。
- 微信手机号绑定接口只接受原生 `getPhoneNumber` 返回的一次性动态 code，服务端通过微信接口取得平台校验号码；不接受客户端直接提交手机号。动态 code、access token 和微信原始响应不落库、不写日志，绑定冲突只返回稳定错误码，不泄露号码或账号归属。
- 当前居民资料只向本人返回已绑定的完整手机号，用于新增地址的本地内存预填；后台居民列表/详情只返回脱敏号码和绑定状态，不返回 OpenID、UnionID、完整手机号或地址。
- 收货地址接口只向地址所属居民返回完整姓名、手机号和门牌信息；这些字段不写入日志、本地持久化、截图或测试/计划证据，退出登录时清除客户端内存状态。

## 边界验证

- 请求体、路径、查询参数和微信回调全部在边界使用 Zod 校验。管理后台对超过 512 KiB 的 PNG/JPEG/WebP 图片先在浏览器端缩放/降质量重新编码，但服务端不信任压缩结果，仍执行最终扩展名、MIME、文件头、Base64 和实际大小校验。
- API 只解析 JSON 请求体且全局限制为 1 MiB，不启用表单或 multipart；管理端图片以单文件 Base64 JSON 请求上传，单张解码后最多 512 KiB。上传路由必须先鉴权，再同时校验 PNG/JPEG/WebP 文件扩展名、声明 MIME、文件头、Base64 合法性和实际大小；媒体标识由服务端生成，客户端不能指定对象键。
- 普通接口与登录入口分别限频；入口 Nginx 覆盖客户端传入的 `X-Forwarded-For`，API 只信任受控代理，内存桶有 10,000 来源硬上限。
- 外部 request id 必须符合安全字符与 128 位上限，否则替换为服务端 UUID，不能把任意请求头直接写入日志或审计。
- 管理后台跨域只允许 `CORS_ALLOWED_ORIGINS` 中的精确 Origin；生产值必须是无路径、无通配符的 HTTPS Origin。预检只开放当前 API 方法及 `Authorization`、`Content-Type`、`X-Request-ID`，不得由入口 Nginx 追加 `Access-Control-Allow-Origin: *`。
- JWT 只证明身份；管理员和居民 token 使用不同 `kind` 且不可互用，每次受保护请求仍需重新确认账号状态与资源范围。
- 店铺管理员结构可以预留，但第一版不能获得跨店访问。
- 受控媒体以不可变二进制记录保存，只通过公共只读图片路由按已验证 MIME 返回，并设置 `X-Content-Type-Options: nosniff` 与公开缓存头；不接受 SVG、GIF、HTML 或其他可执行/主动内容，不提供公共上传、覆盖或删除路由。

## 交易安全

- 金额由服务端从商品快照和配送规则重新计算，使用十进制定点数。
- 创建订单使用唯一请求 id；库存条件更新和订单写入处于同一事务。
- 创建请求只保存不含明文地址、手机号和备注的 SHA-256 参数指纹；订单联系信息只存在归属居民可读的地址快照，不进入结构化日志。
- 居民订单列表和详情始终按当前用户隔离；后台列表默认脱敏联系电话，完整地址和电话只在已鉴权管理员详情中按履约需要返回，后台备注不向居民暴露。
- 管理员只能执行状态机明确允许的关闭和履约动作，不能把订单手工改为 `PAID`；取消库存恢复和状态/操作审计必须原子提交。
- 微信支付通知只在 128 KiB 原始请求体、按 `Wechatpay-Serial` 精确选取的平台公钥签名、时间窗口和 APIv3 AES-GCM 解密全部通过后解析业务字段；未知公钥 ID 或 ID/签名错配直接拒绝，再验 AppID、商户、订单号、交易号、币种、金额和当前状态，最后以通知 id 幂等提交。
- 支付私钥、APIv3 key 与“平台公钥 ID -> PEM”映射只从环境读取；当前主动公钥 ID 必须存在于映射中，轮换期间最多并存 8 把且不得逐把试验签名。通知原文不入库，审计仅保存 SHA-256 摘要、外部通知 id、微信交易号和稳定处理结果。
- 小程序不能提交金额、OpenID、商户号或支付成功状态；客户端 `requestPayment` 结果不是交易真相，订单支付状态只来自可信回调或服务端查单。
- 居民取消、管理员关闭和超时任务在释放库存前必须按商户订单号查单；只在微信明确 `NOTPAY` 并关单成功或已为 `CLOSED` 时提交取消，`SUCCESS` 必须走统一入账，身份/金额不符或外部结果未知时保留库存和可重试审计。
- 居民退款请求不接收金额、交易号或成功状态；服务端从唯一成功支付记录复制全额 Decimal，一单只允许一笔退款，审核和重试始终复用固定商户退款单号。
- 退款通知先验平台签名与时间窗口并解密，再核对商户、原订单号、交易号、商户退款号、微信退款号、币种和金额；只有可信 `SUCCESS` 可原子恢复一次库存和扣回销量，通知原文不落库。
- 订阅授权弹窗只能由小程序内用户点击触发；客户端报告只用于保守预占，不能伪造订单事件。服务端只向订单所属居民的 OpenID 发送已配置模板，模板数据不含姓名、手机号、门牌、备注、token 或 OpenID。
- 订阅发送不记录 access token、OpenID 或微信原始响应；只保存模板 ID、稳定业务事件、状态、尝试次数和归一化错误。外部结果未知不得自动重试，避免重复消息形成骚扰。
- 后台提醒内容限于店铺、订单号、商品名和库存数，列表/声音轮询不返回居民联系方式或地址；提醒已读和解决状态不得反向写订单、退款或库存。
- 后台操作日志只允许状态、资源名称、排序、范围 ID、配送配置、价格和库存等白名单字段进入 before/after；手机号、地址、收件人、微信身份、Token/密钥、备注、说明和详情字段在写入前删除，说明文字中的完整手机号和 Bearer 凭证同样脱敏。
- 操作日志列表不返回 before/after，详情响应再次执行相同去敏；独立 request id 用于追踪且不拼入说明文字。生产 API 只读，不提供更新或删除日志的路径。
- 订单、支付、退款、状态日志和操作日志禁止物理删除。

## 外部与破坏性动作

- 生产部署、真实支付/退款、发送订阅消息、清空数据库和轮换密钥必须显式批准。
- 测试默认使用 sandbox、fixture 或本地容器；不把真实用户数据复制到开发环境。
- 新依赖在 active plan 说明用途；安全相关变更必须带滥用或越权验证。

## 依赖审计基线

- 2026-07-17 的根锁 `npm audit --omit=dev` 结果仍为 0 critical、9 high、14 moderate、11 low，主要来源是官方 DCloud 固定编译批次及 Prisma CLI 传递依赖；`npm run release:audit` 因 high 返回失败并保持生产 `NO-GO`。
- API 使用 `server/runtime/package-lock.json`，后台构建使用 `admin-web/package-lock.json`，两份隔离锁在线审计均为 0 漏洞；API 运行镜像只安装 105 个生产包且不包含 Prisma CLI，后台运行镜像只包含 Nginx 与静态产物。
- 不运行会跨大版本降级或破坏 uni-app 精确 peer 的 `npm audit fix --force`；残余项记录在技术债跟踪中，并在上游兼容版本发布或任何公网/生产发布前复审。
- DCloud 开发服务器只绑定本机、只处理受信任源码和素材；生产容器不安装或运行小程序编译器与 Prisma CLI。
